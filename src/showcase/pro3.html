pro3 · React / Remotion 视频化渲染

Remotion 渲染

React 组件即时间线，可接入 Node 渲染与批量出片管线。

一、产品

这是一套基于 Remotion 的可编程成片能力：把营销片、片头片尾、数据看板视频等，从「每次手工剪辑」收敛成「可复用的模板 + 数据驱动渲染」。同一套描述既能在本地或 CI 里批量出片，也能与现有 Node 渲染服务、任务队列对接，适合模板化强、SKU 或活动参数变化频繁的业务。

与通用剪辑软件相比，优势在于版本化、可测、可自动化：时间线以代码与配置表达，变更可 diff、可回滚；与纯 FFmpeg 脚本相比，又保留 React 生态里的组件化与状态管理，降低复杂包装的维护成本。

三、技术方案

编排层：用 Remotion 在 React 中描述帧、序列与合成关系，时间轴与图层以组件树组织，便于拆成可复用片段与主题变量。

渲染与导出：由 Remotion CLI 或自建 runner 调起浏览器内核完成合成，再经 FFmpeg 输出 MP4 等交付格式；可按并发度部署多 worker，与消息队列或调度系统组合做批量任务。

工程化：依赖、目录约定、与 CI/CD 的衔接以仓库 README 为准；若需与云剪辑主链路对齐，可将「参数 JSON → Remotion composition」作为统一入口，由上游任务写入数据、下游只负责渲染与上传。
